# HR Backend API

## Project Overview

This is a backend service for an HR management system. It is built using **Go** and adheres to a clean architecture pattern. The project utilizes **Gin** for HTTP handling, **pgx** for high-performance PostgreSQL interactions, and **sqlc** for type-safe database code generation.

## Tech Stack

- **Language:** Go (1.25.5+)
- **Web Framework:** [Gin](https://github.com/gin-gonic/gin)
- **Database:** PostgreSQL
- **Driver:** [pgx/v5](https://github.com/jackc/pgx)
- **SQL Generation:** [sqlc](https://sqlc.dev/)
- **Configuration:** Environment variables

## Architecture

The project follows a standard Go project layout with a separation of concerns:

- `cmd/api/`: Application entry points.
- `internal/`: Private application code.
  - `config/`: Configuration loading and management.
  - `handler/`: HTTP handlers (Controllers).
  - `middleware/`: Custom HTTP middleware.
  - `model/`: Domain models (often shared or complementary to DB models).
  - `repository/`: Database access layer. Code here is primarily generated by **sqlc**.
  - `service/`: Business logic layer.
- `pkg/`: Public library code (e.g., generic DB connection logic, loggers).
- `migrations/`: SQL migration files for schema management.

## Getting Started

### Prerequisites

- Go 1.25+
- PostgreSQL running locally or accessible via URL.
- `sqlc` CLI installed (for generating DB code).

### Environment Variables

The application relies on environment variables for configuration. Defaults are provided in `internal/config/config.go` for local development.

- `SERVER_PORT`: Port to run the server on (default: `8080`)
- `DATABASE_URL`: PostgreSQL connection string (default: `postgres://localhost/hrdb?sslmode=disable`)

### Build and Run

1.  **Install Tools & Setup Git Hooks:**

    ```bash
    just setup
    ```

2.  **Generate Database Code (if schema changes):**

    ```bash
    just generate
    ```

3.  **Run the Development Server (with hot module replacement):**

    ```bash
    just dev
    ```

4.  **Run Database Migrations:**

    ```bash
    just migrate
    ```

5.  **Re-run Specific Migration:**

    ```bash
    just remigrate <migration_file.sql>
    ```

6.  **Run Tests:**
    ```bash
    just test
    ```

## Development Status & Conventions

- **Current State:** The project structure is scaffolded, and the database layer (migrations + sqlc) is set up. However, `main.go` is currently a placeholder and does not yet fully integrate the `config`, `database`, and `handler` packages.
- **Migrations:** Use `just migrate` to run migrations or `just remigrate <file>` to re-run a specific migration. Both commands trigger hot reload if Air is running.
- **Database Access:** Do not write raw SQL in handlers or services. Add queries to `internal/repository/query/*.sql`, run `just generate`, and use the generated methods in `internal/repository/`.
- **Dependency Injection:** The application aims to use dependency injection (manual wiring in `main.go`) to inject the repository/DB pool into services, and services into handlers.
- **Error Handling:** Use standard Go error handling.
