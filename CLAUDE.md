# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Commands

- **Build API**: `just build` (Outputs to `bin/server`)
- **Run API (Dev)**: `just dev` (with hot module replacement)
- **Run Migrations**: `just migrate` (Runs migrations and triggers hot reload if Air is running)
- **Re-run Specific Migration**: `just remigrate <migration_file.sql>` (Drops tables, removes record, re-applies, triggers hot reload)
- **Run Tests**: `just test`
- **Run Single Test**: `just test-one ./path/to/pkg TestName`
- **Generate SQL**: `just generate` (Run after modifying SQL in `migrations/` or `internal/repository/query/`)
- **Install Tools & Git Hooks**: `just setup`
- **Lint**: `just lint`

## Architecture

- **Entry Points**:
  - `main.go`: Primary API server entry point. Connects to DB, initializes repositories/services/handlers, and starts Gin server.
  - `cmd/migrate/main.go`: Custom database migration tool.
- **Internal Structure** (`internal/`):
  - `config/`: Configuration loading via environment variables.
  - `handler/`: HTTP handlers (Gin controllers).
  - `middleware/`: HTTP middleware (Auth, CORS, Role-based access).
  - `model/`: Domain models and shared structs.
  - `repository/`: Database access layer. _Auto-generated by sqlc_ from `internal/repository/query/*.sql`.
  - `service/`: Business logic layer.
- **Public Packages** (`pkg/`):
  - `database/`: Database connection logic (pgx pool).
  - `logger/`: Logging utilities.
- **Data Layer**:
  - `migrations/`: SQL schema definitions.
  - `internal/repository/query/`: SQL queries used by `sqlc` to generate Go code.
  - `sqlc.yaml`: Configuration for `sqlc`.

## Development Conventions

- **Database Access**: Strictly use `sqlc`.
  1. Modify `migrations/*.sql` for schema changes.
  2. Add/Edit queries in `internal/repository/query/*.sql`.
  3. Run `just generate`.
  4. Use generated methods in `internal/repository/`.
  - Do NOT write raw SQL in handlers or services.
- **Dependency Injection**: Dependencies (DB pool, Repositories, Services) are manually wired in `main.go` and injected into constructors.
- **Configuration**: Managed via environment variables (defaults in `internal/config/config.go`).
- **Error Handling**: Standard Go error handling patterns.
- **Testing**: Use `testify` for assertions.
